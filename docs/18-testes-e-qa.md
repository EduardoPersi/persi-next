# 18 — Testes e QA

## Objetivo

Garantir qualidade antes de cada publicação.

## Tipos de teste

- Unitários
- Integração
- E2E
- Testes manuais

## Fluxo de QA

1. Executar lint
2. Executar build
3. Validar responsividade
4. Testar navegação
5. Testar carrinho
6. Testar checkout
7. Validar SEO
8. Revisar desempenho

## Testes obrigatórios

### Home

- Banner
- Busca
- Categorias
- Cards

### Produto

- Galeria
- Variações
- Estoque
- Compra

### Carrinho

- Adicionar
- Remover
- Alterar quantidade
- Cupom

### Checkout

- Endereço
- Frete
- Pagamento
- Pedido

## Performance

Validar:

- LCP
- CLS
- INP
- TTFB

## Checklist Final

- Sem erros de console
- Build aprovada
- Lighthouse satisfatório
- Links funcionando
- Acessibilidade revisada
