# 08 — Git, GitHub e Codex

## Fluxo

1. Atualizar branch.
2. Implementar.
3. Testar.
4. Commit.
5. Push somente autorizado.

## Commits

Padrão:

feat(...)
fix(...)
docs(...)
refactor(...)
style(...)
test(...)

## Regras

- Não fazer push automático.
- Não apagar branches.
- Não usar reset --hard sem autorização.
- Nunca versionar secrets.

## Codex

Antes de alterar arquivos:

- Ler AGENTS.md.
- Ler documentação em docs/.
- Entender o contexto.
- Alterar somente o necessário.

## Revisão

Sempre conferir:

- lint
- build
- responsividade
- acessibilidade
- performance

## Checklist

- Código limpo.
- Sem erros TypeScript.
- Sem console.log desnecessário.
- Componentes reutilizados.
- Documentação atualizada quando necessário.
